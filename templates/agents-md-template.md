## Rules for Future Agents

### On Logos
- `01_logos/primary_v1.png` is the canonical primary logo. Treat it as locked.
- All new assets should use `primary_v1.png` as a visual reference for color and style.
- To generate a new logo variant: use `gpt-image-2` edit mode with `primary_v1.png` as `--image_url` reference.
- Never regenerate `primary_v1.png` without explicit user approval.

### On Colors
- Always read `06_color/tokens.json` before specifying any color in prompts or code.
- Use the exact HEX values from tokens.json. Do not approximate or substitute.
- If a new color is needed that isn't in tokens.json, add it there first, then use it.

### On Versioning
- `_v1` files are approved. New versions use `_v2`, `_v3` suffix — never overwrite.
- Version increment on any regeneration: `primary_v1.png` → `primary_v2.png`, etc.
- Only the highest approved version is "current".

### On New Asset Generation
- Read `references/prompt-patterns.md` before building any fal.ai prompt.
- Generate 3 variants for any new logo/patch/sticker → select best → version it.
- Always write a `.meta.json` sibling file with `{prompt, model, date}`.

### On the Brand Voice
- Read Brand.md → "Verb-Trio" section before writing any copy.
- The tone of voice section in Brand-Booklet.md defines what words are banned.
